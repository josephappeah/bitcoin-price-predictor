Long Short-Term Memory (LSTM) Module

HOW IT WORKS


LSTM stands for Long-short term memory and these type of neural networks introduce special memory gates which control the flow of information in the network and prevent exponential decay of the loss function by means of the control gates. The LSTM model developed for this project took in the best 17 features after performing Recursive Feature Elimination with Logistic regression and Extra trees regressor on 23 features. The last 560 days of Bitcoin data was extracted from Quandl.com, out of which the first 500 days were used for training. The next 30 days were used for validating the model to prevent overfitting and the last 30 days were used to test the prediction prowess of the model. The LSTM consisted of 2 layers, 400 memory cells and was trained for 270 Epochs. Figure 7 shows the test results of the LSTM model for the past 30 days.


Neural networks are critical to machine intelligence as they allow the machines to interpret the data on their own. However, regular feedforward neural networks do not do so well when asked to form a context with past data. To tackle this problem, a different form on neural network is deployed, called the Recurrent neural network (RNN). Rather than each ‘neuron’ taking a fresh input, a neuron in RNN additionally takes the output from previous neurons as part of it’s input. This allows it to learn across sequence of data. However, RNNs are not good at remembering output of neurons after a limited number of time-steps back in the sequence.

POLYNOMIAL REGRESSION Module

HOW IT WORKS


The initial feature selection algorithm used to train our predictive models was originally a greedy algorithm known as Recursive Feature Elimination (RFE). This algorithm selected features by recursively iterating over the set of all features, and removing the ones that provided the least impact to the cost function, until the desired number of features is met. This algorithm is very brute force and is time costly. We propose a more robust algorithm known as Principal Component Analysis (PCA), which chooses k-features with the greatest variability, essentially finding k-dimensional feature vectors with the largest variance.


Originally, we were able to show training accuracy of our Polynomial Regression model by using a single feature, namely blockchain size, and training the model over 365 days. We have expanded on this considerably by adding multi-feature data capability, meaning that our model incorporates more than just blockchain size but also other features such as hashrate, number of transactions, etc. We used principal component analysis to choose from 13 different features, and found 6 features that had the largest eigenvalues (variability) and met a threshold variation ratio of 0.95 to choose the number of k features. Due to the increased model complexity, having multiple features, and each having polynomial degrees within the hypothesis function, overfitting has become a dominant concern.

COMPARISON TO MOMENTUM TRADING STRATEGIES

HOW IT WORKS


Momentum is calculated as the average of log returns over the rolling window. Positive value with sufficiently high magnitude results in a recommendation to buy; negative value with high magnitude results in a recommendation to sell. Using daily Bitcoin price data, the momentum algorithm learned that the optimal size window. Figure 8 below illustrates cumulative returns for window of four different sizes and shows that a strategies based on either 30 or 120-day momentum works better than others. As we will describe later in the report, this algorithm was used over the same time interval as the other models as a performance benchmark.

Momentum has to be calculated over a specified size of “rolling window”. From a list of various window sizes, find the one that produces the highest results according to backtesting. Save the optimal window size and create a trader class that will fetch data real-time, find out the direction and magnitude of price movement, and make a recommendation to buy or sell Bitcoin.

There are not many resources available to learn about quantitative trading. The industry appears quite secretive about their algorithms.
